What's special about Rust?
It brings the concept of expressions to a systems programming language
So the syntax is more familiar to C-style languages (like C, C++, Java, etc.)
even though the idea of expressions is rooted in functional programming languages.
  

Coming from imperative languages myself, expressions took me a while to get used to.
Now I miss them a lot in other languages.



special
- almost everything is an expression (blocks loops conditionals match statements)
- match expression more powerful than switch statement.
- error handling with expressions (? to propagate error)
- immutability by default and expressions are immutable by default
  mutability is explicit
- type inference: don't have to annotate types
- performant and safe. no null pointer exception in comparison to other languages
- very much a functional concept. Close to Lisp, Haskell, or Scala.
- what sets Rust apart is memory safety in combination with expressions

in contrast to scala and lisp
- in rust even blocks of code enclosed in curly braces are expressions
- match statement returns a value
- control flow statements are expressions
